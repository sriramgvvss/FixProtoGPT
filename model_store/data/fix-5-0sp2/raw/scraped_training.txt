FIX Data Type INT: Sequence of digits without commas or decimals, may include leading sign
FIX Data Type LENGTH: Non-negative integer representing message length
FIX Data Type NUMINGROUP: Number of repeating group entries
FIX Data Type SEQNUM: Message sequence number (positive integer)
FIX Data Type TAGNUM: FIX field tag number (positive integer)
FIX Data Type FLOAT: Sequence of digits with optional decimal point and sign
FIX Data Type QTY: Float quantity value
FIX Data Type PRICE: Float price value (typically 2 decimal places)
FIX Data Type PRICEOFFSET: Float value representing a price offset
FIX Data Type AMT: Float value typically representing a monetary amount
FIX Data Type PERCENTAGE: Float value representing a percentage
FIX Data Type CHAR: Single character value
FIX Data Type BOOLEAN: Y = Yes/True, N = No/False
FIX Data Type STRING: Alphanumeric free format strings
FIX Data Type MULTIPLECHARVALUE: Space-delimited multiple character fields
FIX Data Type MULTIPLESTRINGVALUE: Space-delimited multiple string fields
FIX Data Type CURRENCY: ISO 4217 currency code (3 chars)
FIX Data Type EXCHANGE: ISO 10383 Market Identifier Code (4 chars)
FIX Data Type UTCTIMESTAMP: UTC date/time: YYYYMMDD-HH:MM:SS.sss
FIX Data Type UTCTIMEONLY: UTC time only: HH:MM:SS.sss
FIX Data Type UTCDATEONLY: UTC date only: YYYYMMDD
FIX Data Type LOCALMKTDATE: Local date: YYYYMMDD
FIX Data Type MONTHYEAR: Month-year: YYYYMM or YYYYMMDD or YYYYMMwN
FIX Data Type DATA: Raw data with separate length field
FIX Data Type XMLDATA: XML-encoded data

FIX Component StandardTrailer: Required trailer for all FIX messages. Fields: CheckSum
FIX Component Instrument: Identifies a security. Fields: Symbol, SymbolSfx, SecurityID, SecurityIDSource, SecurityType, MaturityMonthYear, PutOrCall, StrikePrice, OptAttribute, SecurityExchange, Currency
FIX Component OrderQtyData: Specifies the quantity of an order. Fields: OrderQty, CashOrderQty
FIX Component Parties: Identifies parties in a trade. Fields: NoPartyIDs, PartyID, PartyIDSource, PartyRole
FIX Component SpreadOrBenchmarkCurveData: Spread or benchmark curve data. Fields: 218, 220, 221, 222, 662, 663, 699
FIX Component CommissionData: Commission data. Fields: 12, 13, 479, 497

# FIX Protocol Concepts
The FIX (Financial Information eXchange) protocol is an electronic communications protocol for international real-time exchange of securities transaction information.
FIX 5.0 SP2 is the latest stable version. It uses FIXT 1.1 as the session layer protocol, separating application and session concerns.
In FIX 5.0, the session protocol (FIXT.1.1) is decoupled from the application protocol (FIX 5.0 SP2). BeginString(8) is always FIXT.1.1, and ApplVerID(1128) indicates the application version.

# Message Structure
Every FIX message consists of: StandardHeader + Body + StandardTrailer.
The StandardHeader always begins with BeginString(8), BodyLength(9), MsgType(35).
The StandardTrailer always ends with CheckSum(10).
Fields are delimited by SOH (ASCII 01) in production, often shown as | in documentation.

# Session Layer (FIXT 1.1)
Session messages handle connection management: Logon(A), Logout(5), Heartbeat(0), TestRequest(1), ResendRequest(2), SequenceReset(4), Reject(3).
Each side maintains an independent outgoing sequence number, starting at 1.
The Logon message must be the first message sent by each side after TCP connection.

# Order Lifecycle
New orders are submitted via NewOrderSingle(D). The sell-side responds with ExecutionReport(8).
Orders can be modified via OrderCancelReplaceRequest(G) or canceled via OrderCancelRequest(F).
If a cancel or modify is rejected, OrderCancelReject(9) is returned.
The ExecType(150) field in ExecutionReport indicates what happened: F=Trade, 0=New, 4=Canceled, 8=Rejected.
OrdStatus(39) tracks the cumulative state: 0=New, 1=PartiallyFilled, 2=Filled, 4=Canceled, 8=Rejected.

# Market Data
MarketDataRequest(V) subscribes to market data. SubscriptionRequestType(263): 0=Snapshot, 1=SnapshotAndUpdates.
MarketDataSnapshotFullRefresh(W) provides a complete book. MarketDataIncrementalRefresh(X) provides updates.
MDEntryType(269): 0=Bid, 1=Offer, 2=Trade, 4=OpeningPrice, 5=ClosingPrice.

# Key Differences: FIX 4.x vs FIX 5.0 SP2
FIX 4.x uses BeginString like 'FIX.4.2'. FIX 5.0 SP2 uses BeginString 'FIXT.1.1' with ApplVerID(1128)=9.
FIX 5.0 introduced the Parties component block for identifying counterparties.
ExecTransType(20) was deprecated in FIX 5.0. ExecType(150) values were rationalized.
FIX 5.0 SP2 added enhanced derivatives support, strategy parameters, and improved market data.
